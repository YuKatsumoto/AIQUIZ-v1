# Phase 1 セットアップ手順（AIQUIZUE プロジェクト）

**前提**: Unreal Editor の言語は**日本語**にしています（**編集 → エディタの設定 → 一般 → 言語**で「日本語」を選択）。

Phase 1 の C++ 実装はプロジェクトに追加済みです。以下の手順でエディタでブループリント・レベルを用意するとプレイできます。

---

## 1. ビルド（詳しく）

Phase 1 の C++ を動かすには、プロジェクトを一度ビルドしてコンパイルする必要があります。

### 前提

- **Unreal Engine 5**（5.4 / 5.5 / 5.7 など）がインストール済みであること
- **Visual Studio 2022**（「C++ によるゲーム開発」ワークロード）が入っていること  
  - エディタからコンパイルするだけなら、.uproject に紐づくエンジンが自動でコンパイラを探します

### いつ「Visual Studio 用プロジェクトファイルを生成」を使うか

次のようなときは、**.uproject を右クリック → 「Visual Studio 用プロジェクトファイルを生成」** を実行してください。

- 初めて C++ を追加した／ソースを大幅に変えた
- `Source` フォルダ内の .h / .cpp を追加・削除した
- `.Build.cs` を編集した（モジュール追加など）
- ソリューションを開いたら新しいクラスが認識されていない

実行すると、プロジェクト名 `.sln` が .uproject と同じフォルダに（または指定した場所に）生成されます。

---

### 方法 A: エディタからビルド（手軽）

1. **AIQUIZUE.uproject** をダブルクリックして Unreal Editor を起動
2. メニュー **ツール → コンパイル**（または **Ctrl+Alt+F11**）で C++ をビルド
3. 初回やソース変更後は「モジュールの再コンパイル」が走り、数十秒〜数分かかることがあります
4. 成功すると右下に「コンパイルが完了しました」などと表示されます

**Live Coding** を使う場合:

- **Ctrl+Alt+F11** でコンパイルすると、エディタを閉じずに変更が反映されます（実行中でも再コンパイル可能な範囲で反映されます）
- Live Coding でクラッシュする場合は、**エディタを閉じてから Visual Studio でフルビルド → エディタを開き直す**と安定します

---

### 方法 B: Visual Studio からビルド

1. **Visual Studio 用プロジェクトファイルを生成** を実行したあと、生成された **AIQUIZUE.sln** を開く
2. 上部の構成で **Development Editor**、プラットフォーム **Win64** を選択
3. メニュー **ビルド → ソリューションのビルド**（または **F7**）
4. ビルドが終わったら Unreal Editor を起動（.uproject をダブルクリックしてよい）

デバッグするときは、Visual Studio の **デバッグ → デバッグの開始**（F5）で起動するか、エディタ側の **プレイ** に「プロセスにアタッチ」でアタッチする形が一般的です。

---

### 方法 C: コマンドラインでビルド（自動化・CI 向け）

エンジンの **Build\BatchFiles** にある `Build.bat` を使います。エンジンのパスは環境に合わせて書き換えてください。

```batch
REM 例: UE 5.7 が C:\Program Files\Epic Games\UE_5.7 にある場合
cd /d "C:\Users\あなたの名前\Desktop\AIQUIZ\AIQuizUE\AIQUIZUE"

"C:\Program Files\Epic Games\UE_5.7\Engine\Build\BatchFiles\Build.bat" ^
  AIQUIZUEEditor Win64 Development ^
  -Project="%CD%\AIQUIZUE.uproject" ^
  -WaitMutex
```

- **AIQUIZUEEditor**: エディタ用ターゲット（ゲーム単体の実行ファイルだけなら `AIQUIZUE`）
- **Win64 Development**: プラットフォームと構成
- 成功すると `Binaries\Win64\` などに実行ファイルができます

---

### ビルド後の確認

- エディタを開き、**コンテンツ ブラウザ** の「C++ クラス」や **クラス ビューア**（ウィンドウ → 開発者ツール → クラス ビューア）で `QuizWall` / `QuizGameMode` / `QuizWidget` / `QuizCharacter` などが表示されれば、C++ は正しくビルドされています
- 表示されない場合は、上記の「Visual Studio 用プロジェクトファイルを生成」をやり直してから再度ビルドしてください

---

### よくあるエラーと対処

| 症状 | 対処 |
|------|------|
| 「HTTP」「Json」が見つからない | `Source/AIQUIZUE/AIQUIZUE.Build.cs` の `PublicDependencyModuleNames` に `"HTTP"`, `"Json"`, `"JsonUtilities"` が入っているか確認。入れたら **Visual Studio 用プロジェクトファイルを生成** をやり直してからビルド |
| 不明な型・クラス | 該当 .cpp の `#include` が正しいか確認。`Variant_Quiz` 内のファイルは `#include "Variant_Quiz/○○.h"` のようにパスを書く |
| ビルドは通るがエディタでクラスが出ない | エディタを完全に終了してから再度ビルドし、.uproject でエディタを開き直す |
| 日本語パスでビルドが変になる | プロジェクトを英数字のみのパス（例: `D:\Projects\AIQuizUE`）に移すと安定しやすい |

## 2. ウィジェット ブループリント（WBP_Quiz）

**C++ が問題文・選択肢用のテキストを自動作成するため、次の 3 ステップだけでよいです。**

1. **コンテンツ ブラウザ** で右クリック → **ユーザーインターフェース** → **ウィジェット ブループリント**
2. 名前を `WBP_Quiz` にする
3. ブループリントを開いたら、**詳細** パネル上部の **クラスの設定**（歯車アイコン）をクリックし、**親クラス** を **QuizWidget**（C++）に変更して **Ctrl+S** で保存

以上で完了です。**テキストやパネルを手動で追加する必要はありません。** プレイ時に C++ が「問題文」「左」「右」の 3 つのテキストを画面上部に自動作成します。

---

**（任意）レイアウトを自分で作りたい場合**

デザイナーで見た目を細かく調整したいときだけ、次のように手動で追加してかまいません。

- **プリミティブ** または **一般** から **テキスト**（Text Block）を 3 つ配置し、**識別子** を `Text_Question` / `Text_ChoiceLeft` / `Text_ChoiceRight` に設定する
- その場合、C++ はそれらをそのまま使い、自動作成は行いません
- 正解／ゲームオーバー用に **パネル** や **テキスト** を追加する場合は、識別子を `Panel_Correct` / `Panel_GameOver` にすると表示の切り替えが有効になります

---

## 3. 壁ブループリント（BP_QuizWall）

1. **コンテンツ ブラウザ** で右クリック → **ブループリント クラス**
2. **すべてのクラスを表示** にチェックを入れ、検索で `QuizWall` を入力し、**QuizWall** を選んで **選択**。アセット名を `BP_QuizWall` にする
3. ブループリントを開いたら **コンポーネント** と **詳細** で次を設定:
   - **WallMesh**: **スタティックメッシュ** を設定（基本形状の「立方体」など）。**スケール** で壁のサイズに（例: X=0.05, Y=8, Z=4）
   - **WallBodyCollision**: **ボックス** の位置・サイズを「壁のドア以外の部分」に合わせる
   - **DoorLeftCollision**: 左ドアの穴の位置・サイズに合わせる（廊下の左側）
   - **DoorRightCollision**: 右ドアの穴の位置・サイズに合わせる（廊下の右側）
   - **Wall Speed**: 好みで（例: 200）
4. 各コリジョン（WallBodyCollision, DoorLeftCollision, DoorRightCollision）の **詳細** で **コリジョン プリセット** を **OverlapAllDynamic** などにして、プレイヤーとオーバーラップするようにする
5. **Ctrl+S** で保存

---

## 4. ゲームモード ブループリント（BP_QuizGameMode）

1. **コンテンツ ブラウザ** で右クリック → **ブループリント クラス**
2. **すべてのクラスを表示** にチェックを入れ、`QuizGameMode` を検索し、**QuizGameMode** を親に選択。名前を `BP_QuizGameMode`
3. ブループリントを開いたら **詳細** パネルで:
   - **Quiz Widget Class** に `WBP_Quiz` を設定
   - **Quiz Subject** = 算数（そのままで可）
   - **Quiz Grade** = 3（そのままで可）
4. **Ctrl+S** で保存

---

## 5. キャラクター ブループリント（BP_QuizCharacter）

1. 既存の Third Person キャラのブループリントを複製するか、**コンテンツ ブラウザ** で右クリック → **ブループリント クラス** で新規作成
2. **すべてのクラスを表示** で `QuizCharacter` を検索し、**親クラス** に **QuizCharacter**（C++）を選択。名前を `BP_QuizCharacter`
3. メッシュ・カメラは Third Person のものを流用してよい。カメラは「廊下の奥」が見えるように **スプリングアーム** や **カメラ** の角度・距離を調整
4. **Ctrl+S** で保存

---

## 6. レベル（廊下）

1. **ファイル → 新規レベル** で新規レベルを作成するか、既存マップを複製
2. **廊下**:
   - 床・左右の壁を、**BSP** の「ボックス」や **基本形状** の「立方体」などで 1 本の廊下にする
   - プレイヤーが立つ「手前」と、壁が現れる「奥」を決める。**奥を +X 方向**にすると C++ の壁の移動と一致する
3. **プレイヤー スタート**: **プレイヤー スタート** をパレットからドラッグし、廊下の手前に配置。キャラの向きが奥（+X）を向くように回転を調整
4. **BP_QuizWall** をコンテンツ ブラウザからレベルに 1 つドラッグし、廊下の**奥**（+X 側）に配置
   - 壁の前向きがプレイヤー側（-X）を向くように回転を調整
   - DoorLeft / DoorRight のボックスが廊下の左・右に来るようにする
5. **ワールド設定** を開く（**ウィンドウ → ワールド設定** または **設定 → ワールド設定**）:
   - **ゲームモードオーバーライド** を `BP_QuizGameMode` に
   - **デフォルト ポーン クラス** を `BP_QuizCharacter` に

---

## 7. デフォルトマップにする（任意）

Phase 1 用のレベルを毎回開く場合:

- **編集 → プロジェクト設定** を開く
- **マップとモード**（または **マップ & モード**）で **デフォルト マップ** と **エディタ起動マップ** を Phase 1 のレベルに設定

## 8. 動作確認

1. クイズ API サーバーを起動（`http://127.0.0.1:8000`）
2. エディタの **プレイ**（**再生**）ボタンで実行
3. 問題文・左・右の選択肢が表示され、壁が手前に迫る
4. **A/D または ←/→** で左右に動き、**正解のドア**に進入 → 「正解」→ 次の問題
5. 不正解のドアや壁に触れる → ゲームオーバー

---

## トラブルシューティング

- **問題が表示されない**: WBP_Quiz の **親クラス** が **QuizWidget** か、テキストの **識別子** が `Text_Question` / `Text_ChoiceLeft` / `Text_ChoiceRight` か確認
- **壁が動かない**: レベルに **BP_QuizWall** が 1 つだけあるか、**ワールド設定** のゲームモードが **BP_QuizGameMode** か確認
- **当たり判定が反応しない**: 壁の 3 つのボックス コリジョンの **コリジョン プリセット** を **Overlap** 系にし、プレイヤーのカプセルと反応するようにする
- **API に接続できない**: サーバーを起動し、`UQuizApiSubsystem` の **Api Base Url** が `http://127.0.0.1:8000` か確認（失敗時はフォールバック問題でプレイ可能）
